# Pre-router skica — server-side deterministički tool dispatch prije Claude-a

Ideja: prije nego što request uđe u Claude agent loop, server pokuša 1:1
match imena kategorije. Ako uspije — direktno vraća odgovor (overview za
parent, search za leaf) bez LLM poziva. Ako ne — fall-through u postojeći
Claude flow.


┌─────────────────────────────────────────────────────────────────┐
│  POST /api/chat                                                 │
│  { "message": "Mobiteli", "channel": "chat", "history": [] }    │
└─────────────────────────┬───────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────────┐
│  PRE-ROUTER  (deterministic, no LLM)                            │
│                                                                 │
│  Step 1 — normalize:                                            │
│    "Mobiteli"  → "mobiteli"                                     │
│    "MOBILNI TELEFONI" → "mobilni telefoni"                      │
│    "Mobiteli!" → "mobiteli"                                     │
│    (lowercase, strip diacritics?, strip punct, trim)            │
│                                                                 │
│  Step 2 — exact name match against active categories:           │
│    by_name_norm = { "mobiteli": 151,                            │
│                     "mobilni telefoni": 175,                    │
│                     "maske za mobitele i dodaci": 394, ... }    │
│                                                                 │
│  Step 3 — guard: only bare category-name queries                │
│    if len(words) > 3       → SKIP (kvalifikovan upit)           │
│    if has_price_token      → SKIP ("do 500 KM")                 │
│    if has_brand_keyword    → SKIP                               │
│                                                                 │
│  Step 4 — dispatch:                                             │
│    cid = by_name_norm.get(normalized_message)                   │
│    if cid is None     → FALL-THROUGH to Claude                  │
│    if is_leaf(cid)    → call handle_search_products(category_id=cid)
│    if is_parent(cid)  → call handle_category_overview(parent_id=cid)
└─────────────────────────┬───────────────────────────────────────┘
                          │
              ┌───────────┴───────────┐
              ▼                       ▼
        MATCH                    NO MATCH
        (bare name)              (kvalifikovan / ambiguous)
              │                       │
              ▼                       ▼
     ┌─────────────────┐    ┌──────────────────────┐
     │ handle_search   │    │ Claude agent loop    │
     │ _products(cid)  │    │ (existing flow)      │
     │   ili           │    │   - history          │
     │ handle_category │    │   - tool dispatch    │
     │ _overview(cid)  │    │   - iteracije        │
     └────────┬────────┘    └──────────┬───────────┘
              │                        │
              └────────────┬───────────┘
                           ▼
                ┌──────────────────────┐
                │  ChatResponse        │
                │  + _trace polje      │
                │  + via="prerouter"   │
                │     ili "claude"     │
                └──────────────────────┘


# Pseudokod (app/prerouter.py — novi modul, ~50 LOC)

def try_pre_route(message: str) -> dict | None:
    """
    Vrati ChatResponse dict ako match, None ako fall-through u Claude.
    """
    norm = _normalize(message)

    # Guards: skip kompleksne upite — to nije bare category name
    if len(norm.split()) > 3:
        return None
    if _looks_like_qualified(norm):  # cijena, brend, model code...
        return None

    cid = _name_to_id(norm)  # eager-cached dict iz categories.csv
    if cid is None:
        return None

    if _is_leaf(cid):
        return handle_search_products(category_id=cid, top_k=10)
    if _is_parent(cid):
        return handle_category_overview(parent_id=cid)

    return None  # cat exists ali nije ni leaf ni valid parent


# Test plan — kako provjeravamo da pre-router ne razbija postojeće

1. Eval cold scenario:
   - "Mobiteli" → match na 151 (parent) → overview. OVERVIEW_PASS, no LLM call.
   - "Mobilni telefoni" → match na 175 (leaf) → search. EXACT_PARENT, no LLM call.
   - Verifikacija: oba scenarija sada PASS u skripti BEZ konteksta — to je 100%
     deterministic dio. Ono što je prije bilo prompt issue, sad je server logika.

2. Eval kvalifikovan scenario:
   - "gaming miš do 100 KM" → guard hvata "do 100" → SKIP → Claude. Postojeće
     ponašanje očuvano.
   - "Apple iPhone 17" → 3 riječi ali ima brend → SKIP → Claude.

3. Eval ambiguous:
   - "nešto za gaming" → nema name match → SKIP → Claude.

4. A/B compare:
   - Run before pre-router (label v1-pre-router-off) i run after
     (label v2-pre-router-on). Verdict bi trebao da bude: bare imena prelaze
     iz OVERVIEW_WRONG / FAIL u PASS, kvalifikovani upiti ostaju isti.


# Otvoreno za odluku

- Diacritics: stripamo ili ne? "Mobiteli" vs "mobitéli" (ne realno, ali...).
  Preporuka: ne strip-amo BHS dijakritike u prvoj verziji — kategorije već
  imaju "Graficke" (bez č), pa ostavi as-is.

- Pluralizacija: "mobitel" vs "mobiteli"? Trenutno ne radimo lemmatizaciju —
  exact match. Ako Ivan kuca "mobitel" — SKIP → Claude. To je OK za prvi
  korak; lematizaciju dodajemo kasnije ako se javi kao pattern u dashboard
  Sessions tabu.

- Šta sa "Maske za mobitele" gdje ime sadrži više riječi — guard "len ≤ 3"
  je preuzak. Treba relaksirati za poznata imena kategorija (length kategoriji
  imena, ne fiksiranog 3).

- Pre-router može vratiti i `via="prerouter"` u trace tako da u dashboard-u
  vidimo koliko request-a ide kroz pre-router vs Claude. Već treba u
  ChatResponse.
